📡 API ENDPOINTS
═══════════════════════════════════════════════════════════════════════════════

POST /api/aws/query
  ├─ Query scheme assistance
  ├─ Language support (default: en)
  └─ Returns: {answer, sources, metadata}

POST /api/aws/query/multilingual
  ├─ Auto-detect language
  ├─ Response in target language
  └─ Returns: {answer, sources, detected_lang}

GET /api/aws/health
  ├─ Check AWS services status
  └─ Returns: {bedrock, dynamodb, s3, credentials}

GET /api/aws/info
  ├─ Service information
  └─ Returns: features, endpoints, AWS config

🔑 ENVIRONMENT SETUP
═══════════════════════════════════════════════════════════════════════════════

Create server/.env with:

# AWS Credentials
AWS_REGION=us-east-1
AWS_ACCESS_KEY_ID=your_key
AWS_SECRET_ACCESS_KEY=your_secret

# Services
BEDROCK_MODEL_ID=anthropic.claude-3-haiku-20240307-v1:0
DYNAMODB_QUERIES_TABLE=sahaay-queries
DYNAMODB_USERS_TABLE=sahaay-users
DYNAMODB_SAVED_SCHEMES_TABLE=sahaay-saved-schemes
AWS_S3_BUCKET=sahaay-documents

# Server
NODE_ENV=development
PORT=5000

🆘 TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

Problem: "npm install fails"
Solution: Node.js 18+ required. Check: node --version

Problem: "AWS credentials not found"
Solution: 
  1. Set AWS_ACCESS_KEY_ID and AWS_SECRET_ACCESS_KEY in .env
  2. Or run: aws configure
  3. Or use IAM role (on EC2/Lambda)

Problem: "Cannot connect to AWS services"
Solution:
  1. Check credentials are valid
  2. Check internet connection
  3. Check AWS region is correct
  4. Check IAM permissions

Problem: "Port 5000 already in use"
Solution: PORT=3000 npm start (use different port)

See IMPLEMENTATION-GUIDE.md for more troubleshooting!
